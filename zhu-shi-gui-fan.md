## 注释规范说明

> 为使代码结构更清晰, 统一要求使用`// MARK:`对代码进行结构划分

标注划分的条件是, 结构内的方法数超过2个. 若只有1个方法, 可不添加

1. \[建议\] 使用 `// MARK: - 描述信息` 的结构, 描述一级分组
1. \[建议\] 如果在分组方法中需要继续划分分组, 则采用逐级添加的结构
    * 一级标注, 使用 `// MARK: - 描述信息` 结构
    * 二级标注, 使用 `// MARK: -- 描述信息` 结构
    * 三级标注, 使用 `// MARK: --- 描述信息` 结构
    * 以此类推, 注意尽量不要用到第四级标注

> 注意 : `[建议]` 不要使用 `MARK://` 作为代码注释, 代码注释使用 `///`

### `[建议]` 文档注释风格
> 每一行文档注释前面使用 `///` 符号。不使用 Javadoc 的风格`/** ... */`。

```
✅，XCode可识别的注释
/// Returns the numeric value of the given digit represented as a Unicode scalar.
///
/// - Parameters:
///   - digit: The Unicode scalar whose numeric value should be returned.
///   - radix: The radix, between 2 and 36, used to compute the numeric value.
/// - Returns: The numeric value of the scalar.
func numericValue(of digit: UnicodeScalar, radix: Int = 10) -> Int {
  // ...
}

❌, Javadoc的风格
/**
 * Returns the numeric value of the given digit represented as a Unicode scalar.
 *
 * - Parameters:
 *   - digit: The Unicode scalar whose numeric value should be returned.
 *   - radix: The radix, between 2 and 36, used to compute the numeric value.
 * - Returns: The numeric value of the scalar.
 */
func numericValue(of digit: UnicodeScalar, radix: Int = 10) -> Int {
  // ...
}
```

### 一句话总结
> 文档注释开头应该是一句简短的总结性介绍, 应避免冗余的信息。
> 如果还有一些细节信息需要补充，换行后重起一段话。

```
/// 视图的背景色 ✅
/// 该属性是视图的背景色 ❌（应避免冗余的信息）
var backgroundColor: UIColor


/// 返回数组中的所有数字总和
///
/// - Parameter numbers: 保存数字的数组
/// - Returns: 所有数字的总和
func sum(_ numbers: [Int]) -> Int {
  // ...
}
```

### 参数, 返回值, 抛出的错误
如果文档要说明参数、返回值、抛出的错误，使用 `Parameter(s)`、`Returns`、`Throws` 标签。
如果需要换行，在下一行前面缩进两个空格。

> 注意： 强烈建议使用 Xcode 的快捷键 Command + Option + / 根据函数签名生成文档模板

```
/// Returns the output generated by executing a command.      ✅
///
/// - Parameter command: The command to execute in the shell environment.      ✅ (单个参数使用 Parameter)
/// - Returns: A string containing the contents of the invoked process's  
///   standard output.        ✅ （需要换行，在下一行前缩进两个空格)
func execute(command: String) -> String {
  // ...
}


/// Returns the output generated by executing a command with the given string
/// used as standard input.
///
/// - Parameters:        ✅ (多个参数使用 Parameters)
///   - command: The command to execute in the shell environment.
///   - stdin: The string to use as standard input.      ✅ (参数前缩进两个空格，以 - 开头)
/// - Returns: A string containing the contents of the invoked process's
///   standard output.
func execute(command: String, stdin: String) -> String {
  // ...
}
```

### XCode 支持的 Markup 格式
> 写文档的时候可以灵活的使用苹果的 `markup` 格式。这些标记可以被 `Xcode` 识别，在阅读的时候可以被渲染出不同的格式。

可参阅官方文档 [Markup Fromatting Reference](https://developer.apple.com/library/archive/documentation/Xcode/Reference/xcode_markup_formatting_ref/index.html#//apple_ref/doc/uid/TP40016497-CH2-SW1)

有一些和 markdown 格式很接近。列出几个常用的：
* 段落之间使用以 `///` 开头的单行分隔。
* 两个`*`号包裹表示表示斜体。
* 两个`**`号包裹表示粗体。
* 两个 \` 号包裹表示代码。
* 两个\`\`\`号包裹表示代码块（多行代码）
